New members to ADD (RESEARCH Code Examples lines 421-451 — canonical extension spec):
```solidity
mapping(uint256 agentId => mapping(uint256 signalId => bytes)) private _signalData;     // for signalAt() (D-33)
mapping(uint256 agentId => mapping(uint256 signalId => bool)) public invalidated;

event SignalDecoded(
    uint256 indexed agentId, uint256 indexed signalId,
    address indexed tokenIn, address tokenOut, uint256 amountIn, uint256 minAmountOut, uint24 fee
);
event SignalInvalidated(uint256 indexed agentId, uint256 indexed signalId, string reason, uint64 timestamp);

function invalidateSignal(uint256 agentId, uint256 signalId, string calldata reason) external nonReentrant {
    Source storage s = sources[agentId];
    if (!s.registered) revert SourceNotRegistered(agentId);
    if (s.owner != msg.sender) revert NotSourceOwner(agentId, msg.sender);
    invalidated[agentId][signalId] = true;
    emit SignalInvalidated(agentId, signalId, reason, uint64(block.timestamp));
}

function signalAt(uint256 agentId, uint256 signalId) external view returns (bytes memory) {
    return _signalData[agentId][signalId];
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator EOA → SourceRegistry.recordSignal/invalidateSignal | Only the registered source owner may write/invalidate |
| arbitrary bytes → recordSignal decode | Payload is opaque; decode must not let malformed bytes revert recording |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-03 | Tampering | unauthorized recordSignal / invalidateSignal (someone writes for the agent) | mitigate | Existing NotSourceOwner gate (msg.sender == sources[agentId].owner, D-08) carried verbatim into invalidateSignal; covered by test_invalidateSignal_revertsForNonOwner |
| T-1-04 | Tampering | reentrancy on state-changing registry fns | mitigate | `nonReentrant` on recordSignal + invalidateSignal (D-09, existing pattern) |
| T-1-05 | DoS | malformed signal bytes revert recordSignal (griefs the agent's own writes) | mitigate | Defensive guarded decode — non-conforming bytes skip the typed emit but still record (D-07 opacity); covered by test_recordSignal_opaquePayloadStillRecords |
</threat_model>
